# RustLike

A minimal roguelike prototype built in Rust using the Bevy engine.  
Features procedural dungeons, tile‑based movement, and a simple state machine; designed as a sandbox for experimenting with ECS game design.

## Getting Started

```bash
# clone
git clone https://github.com/ReedPosehn/RustLike.git
cd RustLike

# generate placeholder assets (requires Pillow)
python generate_assets.py

# install dependencies & run
cargo run
```

Assets are under `assets/` and are generated by `generate_assets.py` using Pillow; you can re-run it to modify or inspect sprites.

## Features

- Procedural dungeon generation with random rooms and corridors
- Tile-based player movement with collision (WASD / arrow keys)
- Hub ↔ dungeon state transitions via staircase tiles
- Global scale factor for easy zoom adjustment
- Pixel-art sprite assets for player classes, enemies, and tiles

## Roadmap

### ✅ Phase 1 – Prototype (complete)
- Project setup: Cargo + Bevy, window config
- Tilemap system with hub and dungeon layouts
- Procedural dungeon generator (rooms, corridors, stairs)
- Player rendering and basic WASD movement with collision
- State transitions between hub and dungeon

### 🔧 Phase 2 – Improve Movement & Collision
- Revisit hitbox strategy: `Collider` component with configurable size/offset
- Sliding movement along walls instead of a full stop
- Diagonal movement with consistent speed
- Consider continuous collision detection for smoother feel

### 🏗️ Phase 3 – Architecture & Developer Experience
- Split `main.rs` into modules (`player.rs`, `map.rs`, `systems.rs`, `state.rs`)
- Adopt Bevy's first-class `States` API to replace manual `match game_data.state` guards
- Use `OnExit`/`OnEnter` schedules for tile despawn/spawn (removes the transition cooldown hack)
- Seed-based dungeon generation for reproducibility and future save/load support
- Automated tests for map generation and tile-coordinate helpers

### ⚔️ Phase 4 – Enemies & Combat
- Enemy spawning in dungeon rooms (goblin, orc, skeleton sprites already in assets)
- Simple AI: random walk or player-chase behaviour
- Melee combat: health component, damage, attack-when-adjacent
- Death, despawn, and kill counter
- Multiple player classes (mage, paladin, rogue sprites already present)

### 🎮 Phase 5 – Gameplay Mechanics
- Inventory and item pickup
- Procedural level features: keys/locks, themed rooms
- Dungeon depth and increasing difficulty

### ✨ Phase 6 – Polish & UI
- HUD: health bar, inventory, message log
- Sound effects and music
- Save/load system
- Walk cycle, idle, and attack animations
- Permadeath and roguelike meta-progression
